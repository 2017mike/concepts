Object Destructuring

Don't worry too much about the javascript and console logs in this one, I will put them but I'm going to try to explain it in the HTML.

OK, so let's say we have an object called bigMike. It will look like this:
let bigMike =
{
age: 28,
swag: 'unlimited',
strengths: ['really cool', 'has a tattoo', 'good listener']
}

Well, if we wanted to go and set a variable to be equal to each value, we would have to do this:

let age = bigMike.age
let swag = bigMike.swag
let strengths = bigMike.strengths

But that's the old, boring way to do it. It also takes more lines of code and isn't all that convenient.

With object destructuring, we can set new variables to be equal to any value within the key value pairs by saying this:

let { age, swag, strengths } = bigMike

We now have a variable called age equal to 28,
a variable called swag equal to unlimited,
and a variable called strengths equal to ['really cool', 'has a tattoo', 'good listener'].

Although it looks like we are setting up our bigMike object to be something different, we're actually not. if you console log your object after destructuring it, it will look the same.

Destructuring in .then statements, and some other objects


So while the last example is useful it's probably not the most common way you will see destructuring. You can destructure any object including the the ones you would see after a .then statement. For this example let's use axios and query the pokemon api since we've done that before.


Our axios request would normally look like this:
axios.get('https://pokeapi.co/api/v2/pokemon/kangaskhan')
.then (res => {
console.log(res.data)
let pokemon = res.data
})

Well, we know that res is an object with a KEY of data on it, and a bunch of stuff inside of that VALUE. If we just console logged res, it would look something like this:
{
data: a whole bunch of stuff
}
Since we know we only want the DATA key from the RES object, we can destructure our query to look like this:
axios.get('https://pokeapi.co/api/v2/pokemon/kangaskhan')
.then (({data}) => {
console.log(data)
let pokemon = data
})

Maybe you are wondering, why the parentheses? Shouldn't it just look like this?

axios.get('https://pokeapi.co/api/v2/pokemon/kangaskhan')
.then ({data} => {

well, this is just one of those weird javascript syntax things. this just happens to be the way it's written.


Ok, one last thing here.

we can actually use a colon after the object we destructured to rename it. So, so far we have taken ONLY the data KEY and VALUE from res, but it is still called data. What we can do here is this:
axios.get('https://pokeapi.co/api/v2/pokemon/kangaskhan')
.then (({data: pokemon}) => {
We now have a variable called pokemon that is equal to res.data.
